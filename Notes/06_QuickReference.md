## 常见问题

### Q: 双向链接和普通URL链接有什么区别？
**A:** 
- **双向链接** `[[笔记名]]`：连接同一库中的笔记，支持反向链接
- **URL链接** `[文本](https://example.com)`：指向外部网站，无反向链接

### Q: 如何在Obsidian中快速找到笔记？
**A:** 使用 Ctrl+P (Windows) 或 Cmd+P (Mac) 打开命令面板，输入笔记名称快速搜索。

### Q: 为什么有些链接显示为红色？
**A:** 红色链接表示目标笔记尚不存在。点击链接可快速创建新笔记。

### Q: 如何批量修改标签？
**A:** 在搜索中使用 `tag:#标签名` 查找所有使用该标签的笔记，然后批量编辑。

### Q: 嵌入和链接有什么区别？
**A:**
- **链接** `[[笔记]]`：点击导航到该笔记
- **嵌入** `![[笔记]]`：在当前位置显示该笔记的内容

### Q: 如何在预览模式和编辑模式间切换？
**A:** 按 Ctrl+Shift+! (Windows) 或 Cmd+Shift+! (Mac)

### Q: 标签可以包含空格吗？
**A:** 不可以。标签不能包含空格，但可以用嵌套标签：`#项目/项目A`

### Q: 如何给不同的笔记分组？
**A:** 结合使用文件夹和标签：
- **文件夹**：物理组织方式
- **标签**：逻辑分类方式（支持多个标签）

## 最佳实践

### 命名规范
```
笔记名称：使用清晰、具体的名称
❌ 笔记1、想法、临时
✓ 项目A-需求分析、Python学习笔记、2026-07-07会议记录

嵌套文件夹：3层以内最佳
❌ 项目/分类/子分类/子子分类/笔记
✓ 项目/分类/笔记
```

### 链接策略
```markdown
- 频繁引用的笔记使用双向链接
- 外部资源使用URL链接
- 关键内容使用块引用精确引用
- 避免过度链接导致图谱混乱
```

### 标签使用
```markdown
✓ 创建标签体系：#项目/项目A、#阅读/技术、#待处理
✓ 使用嵌套标签进行多维分类
✓ 定期审查和整理标签
❌ 避免创建过多一级标签
❌ 避免在标签中使用特殊字符
```

### 文件组织
```
文件夹结构示例：
.
├── 项目
│   ├── 项目A
│   ├── 项目B
├── 阅读
│   ├── 技术
│   ├── 非技术
├── 日记
├── 模板
└── README.md
```

### 使用模板
为常见笔记类型创建模板以提高效率：
```markdown
模板内容示例：
---
tags: [模板类型]
date: <% moment(tp.file.creation_date()).format("YYYY-MM-DD") %>
---

# [[标题]]

## 概述

## 详细内容

## 参考链接
```

## 语法速查表

| 功能 | 语法 | 示例 |
|------|------|------|
| 标题 | `# 文本` | `# 一级标题` |
| 加粗 | `**文本**` | `**加粗文本**` |
| 斜体 | `*文本*` | `*斜体文本*` |
| 删除线 | `~~文本~~` | `~~删除~~` |
| 高亮 | `==文本==` | `==高亮==` |
| 链接 | `[文本](URL)` | `[Google](https://google.com)` |
| 内部链接 | `[[笔记]]` | `[[项目计划]]` |
| 图片 | `![描述](路径)` | `![图](image.png)` |
| 嵌入 | `![[笔记]]` | `![[笔记名]]` |
| 代码 | `` `代码` `` | `` `print()` `` |
| 代码块 | ` ```语言\n代码\n``` ` | ` ```python\nprint("hi")\n``` ` |
| 列表 | `- 项` | `- 项目` |
| 有序 | `1. 项` | `1. 第一` |
| 任务 | `- [ ] 任` | `- [ ] 待办` |
| 引用 | `> 文本` | `> 引用` |
| 表格 | `\| 列1 \| 列2 \|` | 见上表 |
| 分割线 | `---` | `---` |
| 标签 | `#标签` | `#项目/A` |
